## 익숙하지 않은 녀석 
java, swift만 사용해오던 나에게 * 은 그저 '곱셈 연산자'로만 인식되어 왔다. 
그런데 왠걸 C++을 사용해야하는 순간이 오자 가장먼저 나의 눈을 어지럽히던 것이 있었는데, 바로 자료형 뒤에 붙은 * 이다.

## 포인터 
말로만 듣던 '포인터' 라는 녀석이었다. 예전부터 선배들이 종종 "포인터가 이해가 안가면 빨리 다른 길을 알아봐라" 하며 겁을 주었던 기억이 있는데 막상 마주해보니 왜 그런말을 해주셨는지 조금은 이해가 된다. 

포인터의 개념을 알아보기 앞서, 우리가 너무나 자연스럽게 사용하고 있는 변수와 메모리에 대해 간단하게 알아보고자 한다. 
```C++
int year = 1992; 
```
위 코드를 보면 우리는 "year 변수에 1992값을 저장한다." 정도로 인식한다. 하지만 메모리의 실제 모습을 고려해보면 단순히 변수 year라는 곳에 저장한다고 하기보다는, **int** 자료형을 갖는 메모리공간의 이름을 **year**로 약속하고, 그곳에 1992라는 값을 저장하는 것으로 보아야한다. 

쉽게 말해, "`변수`는 값을 보유하고 있는 `메모리 공간의 이름`" 이다. 

그렇다면, 이 메모리의 이름 말고 실제 주소는 어떻게 알 수 있을까? 
```C++
&year //1808167260
```
주소연산자 `&`를 사용하면 해당 변수, 즉 해당 메모리의 실제 주소를 알 수 있다. 

이런 못생긴(우리가 보기에) 주소 값을 도대체 어디에 쓰려고 알아내는 것일까? 바로 포인터이다. 

**포인터**는 메모리의 실제 주소를 사용한다. 
```C++
int* ptr = &year;
```
포인터를 처음 접했을 때에는 여러모로 일반 변수와 비슷하게 생겨서인지 정확한 개념이 확 와닿기보다는 혼란스러웠다. 그래서인지 최대한 나의 언어로 이해해보려고 했고, 간단하게 비유, 정리하자면 다음과 같다. 

1. `변수`는 `택배함`과 비슷하다. 
   ```C++
   int a = 12; // a라는 이름을 가진 택배함에 12를 저장!
   ```
2. `포인터`는 `택배함`의 실제 주소가 적힌 운송장과 비슷하다.
   ```C++
   int* ptr = &a; // a택배함의 실제 주소(서울시~...)를 ptr운송장에 기록해둠! 
   ```
3. 역참조`*`는 운송장에 적힌 실제 주소로 직접 가는 행위라고 볼 수 있다.
    ```C++
    *ptr = 13; // a택배함에 13을 저장! 
    ```

이러한 비유적인 표현이 누군가에겐 오히려 혼란을 야기할 수 있지만, 나에겐 이해하는데에 큰 도움이 되었다. 혹시라도 나처럼 포인터형 변수에 대한 개념이 정리가 되지 않았던 사람이 있다면 본 포스팅을 보고 도움이 되었으면 좋겠다.
